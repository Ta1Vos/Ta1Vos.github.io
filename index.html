    The Path of the Dabloons Remastered
    The Path of the Dabloons Remastered

    This is the 10th project, in period 8. This is the last project for the 2nd year. I had to work in a team regarding the SCRUM again.

    For this project I could choose myself what I could make. After some brainstorming I decided to create a remaster for one of my own projects, called The Path Of The Dabloons.

    One of the requirements was that the project had to make use of CRUD and a user/role environment. I've decided that I could combine The Path Of The Dabloons with Symfony, since all the data used in the game can be easily managed in a CRUD system.

    Firstly I established all the documentation for the website (such as the needs analysis, functional design, the release planning, the technical design and the Entity Relationship Diagram). After the documentation was ready I started programming the system:

    • In the 1st sprint I started with the starting pages (home, about me, settings).
    • In the 2nd sprint I programmed simple functionalities such as registering, logging in/out and the entities translated from the Entity Relationship Diagram. Since I was done sooner than expected with these functionalities, I started working ahead for a bit, where I added profile customizations, such as changing your username, email and password.
    • In the 3rd sprint I have focused on member moderation properties, which can be done by users with the moderator role. This included allowing moderators to search for members, temporarily remove/ban members, permanently ban members and changing the data of members in the case it was necessary. If you hadn't noticed yet, this functionality has been based off of the moderation functionality in project FNOJ, only made in Symfony instead of vanilla PHP.
    • In the 4th sprint I've worked on the creator dashboard/admin side of the application, the person who can manage the content found within the application/game. This included creating the dashboard Reads and Deletes. Besides the creator dashboard, the admin is also capable of managing members and moderators.
    • In the 5th sprint I've worked on most Creates and Updates for the entities in the creator dashboard. Besides these functionalities I've also worked on creating the game page where the game will reside when everything is finished.
    • In the 6th sprint I've worked on creating the save file system, which creates a 'player' where all data will be bound to. Besides the save files I've worked on building an API connected to the application, which allows the front-end to request specific data in JSON format, so that there can be a dynamic gameplay without having to constantly refresh the webpage in order to access information. I've also created a rarity generation system so that the RNG system used in the original game, is also (partially) implemented here.
    • In the 7th sprint, the last sprint, I've worked on minor improvements and bug fixes, so that the project would be clear for review by the teachers.
    Programming languages used: PHP/Symfony, HTML, CSS, MySQL

    AEHT

    AEHT - Internship ROC Mondriaan
    AEHT - Internship ROC Mondriaan

    My study has asked me to work for the AEHT during my internship period.
    The AEHT (European Association of Hotel and Tourism Schools) organises a couple of events each year, where many European Hotel and Tourism schools are invited to. The events can be found on the Experiences page.

    My tasks for this internship were:

    • Maintain, update and add new features to an existing application (on app.aeht.eu). Most of my work has been put into this part of my internship.
    • Make sure data given by head-judges (Annual Conference), schools (Christmas In Europe) and the organising school (Youth Parliament) are implemented into the web-application.
    • Promote the app during the events organised by the AEHT, so that the app gets used properly by the participants and visitors of the event.
    • Work on a flutter application that runs the web-application, add a couple of features and make sure this app gets released on the Apple Appstore. This app is mainly used for the events where people can use the app, instead having to search up the web-application in many additional steps.

    A few of the things I have worked on:

    • I have reworked the largest part of the Administrator interface, the part of the website which manages most of the data input by the Admin.
    • I have optimized a lot of code to make some functionalities easier to use as a developer. I've also made it so that a lot of reusable functionalities - where some of which weren't reusable before - can be found back in Symfony Service containers. A few examples would be file management and also dynamic filter generation based on entities and their properties.
    Programming languages used: PHP/Symfony, HTML, CSS, MySQL, Dart/Flutter
